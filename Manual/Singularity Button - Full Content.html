On the Subject of Singularity Button

Oh there is one of this. This should be easy. Wait is there more of itself?

This module consists of a singular round button with a screen at the center. The screen may or may not show a character at a certain time interval. To disarm this module, you must disarm ALL instances of Singularity Button attached to the given casing by interacting with them in a specific way. An incorrect combination of actions on the buttons will cause a strike. The display on Singularity Button may or may not change after a strike. The color around the button's display will not change, however.

Appendix: The Disarm Button

It's there for a reason. And yes, you may not like it at all.

On the back of the module is a red rectangular button that says "DISARM MODULE MANUALLY." When pressed, this will automatically disarm the given Singularity Button only when Singularity Button's are put in a "solve state". Due to strange natures on the "solve detection system" for modules that handle synchronized solves like Singularity Button, all Singularity Button's will flip to the back if bomb does not consist of only Singularity Button due to the fact that there is no way Singularity Button could directly connect to a module that relies on solves without an override. Notable actions that forbid synchronized solves are, reading a stage on major modules that require an input at the end, and inputting an answer on a module before another solve. No time, solve or strike dependent actions are required for the disarm button to be pressed.

Originally, Singularity Button was going to detect if certain modules were present on the bomb and by their conditions. In summary, the effort to set up a system for solve dependent detection system was not worth the time at the end.

Solving 1 Singularity Button

The first one is always simple, right?

The module will display a single digit at the center of the button. In order to disarm 1 Singularity Button, the defuser needs to hold the button at a given time and release it when the conditions are met.

Holding the Button

  • If the button is red, green, or blue, press and immediately release the button when the seconds timer contains the digit on the display. Don't try to press and immediately release too late though, otherwise the module will interpret different time stamps registered for this module.
  • Otherwise, if the button is white or black or gray, hold the button at any time and refer to "Releasing Singularity Button" on this page.
  • Otherwise hold the button when the last digit in the seconds timer is the digit on the display and refer to "Releasing Singularity Button" on this page.

Releasing Singularity Button

Upon holding the button for a short bit, the module will display digits 0 to 9 with 1 digit missing from the sequence. The module will detect that as a hold when this is shown. That digit is referred as the missing digit if the rules require this.

  • If the button is black, release the button when the last digit in the seconds timer is (9 - the missing digit).
  • Otherwise, if the button is white, release the button when the seconds timer contains the digit before you held the button.
  • Otherwise, if the button is gray, release the button when the seconds timer contains the missing digit.
  • Otherwise release when the last seconds digit is the missing digit.

HOWEVER, if blue is the additive color around the button and the button is not gray, the actual displayed digit before holding is (9 - the digit on the display).

Solving 2 Singularity Buttons

Twins! Or the Yin and the Yang of the system.

The 2 buttons will ALWAYS be white and black respectively. In order to disarm both of them, you must input a binary sequence of 16 digits. However, both Singularity Buttons will show a hexadecimal digit one at a time on either button. Both buttons flash white to signify end of sequence. Take notes on where the digits are flashing as this signifies inverted bits in the sequence. The 4-digit hexadecimal is shown from highest to lowest significant digit on the buttons.

If the hexadecimal digit is shown on the white button, the significant bit in the 4 bit array from high to low is inverted in the input sequence, I.E if the flashing digits are in the pattern show from white,black,white,black in this order and one of the flashed digits has a binary representation of 0001 from high to low, the sequence to input for that digit is 1101, low to high. Otherwise, the bit in that array is not inverted.

Now that you got you got the final sequence to input, the defuser can input the bits where 0 represents the black button and 1 represents the white button. Both buttons' actions must be a tap for the module to register. If there are no digits in the serial number or the last digit in the serial number is even, the black button must be tapped at an even time and the white button must be tapped at an odd time. Otherwise, the black button must be tapped at an odd time and the white button must be tapped at an even time. Significant bits must be entered lowest to highest. Inputting the wrong bit or at the wrong time will cause a strike.

The module will stop displaying once the first bit is entered correctly. To clear inputs, hold either button for at least 5 timer ticks or input an incorrect bit.

Solving 4 Singularity Buttons

Simon's Singularity

The 4 buttons are ALWAYS colored Red, Yellow, Green, or Blue. To solve this module, you need to interpret by using other related Simon manuals out from ktane.timwi.de. Correct button presses can change. One button will flash a colored number, where the number is the manual corresponding to the given module. The color from the number refers to the color of the button to start looking for from the external manual. If the desired color to press is NOT red, yellow, green or blue, press the color that is not in the list of the main colors of the module's manual provided. (I.E. 3 G wants a pink button to be pressed but no pink button exists, press the red button, since yellow, blue, and green are present in Tasha Squeals.) If the module's manual requires the position of the given button, it is best assumed to check the other buttons for those. If the color on the module's manual does not exist at all on Singularity Button, skip it, unless after skipping, there are no other colors to press, hold either of the buttons that flashed on this stage and refer to "Holding A Singularity Button MK2" on the next page.

Holding A Singularity Button MK2

Upon holding a button, that button will display colored text which contains a random digit. After releasing the button, resume inputting the rest of the sequence if the desired set of buttons require it.

  1. If the button is displaying text and color inconsistently, release the button any time.
  2. Otherwise, if the button is only displaying text inconsistently grab the color that the button is displaying and refer to the digit on the seconds timer that needs to be present upon releasing the button:
    1. Red: 1
    2. Yellow: 5
    3. Green: 3
    4. Cyan: 7
    5. Blue: 4
    6. Magenta: 4
    7. White: 7

Solving 9 Singularity Buttons

It looks like someone tried to make another Stores module... This time it went all over the place.

The module will consist of button colors reassembling another module however the buttons are scattered. Use the manual from Simon Stores plus this page to grab the value needed to disarm this set. b4 is the target value to disarm this.

The module will display 5 flashes with the flash at XX:X1/X1.XX on the countdown timer being the end of sequence. Steps are applied based on the countdown timer ticking down until the end of sequence where the first step is at XX:X9/X9.XX and the last step is at XX:X3/X3.XX. At XX:X0/X0.XX, colored Singularity Button will show a digit on the display which represents safe digits* to tap when inputting. While the module does not accurately represent where each button is, it is best assumed that swapping colors are not needed to input. Drop decimal answers when dividing.

an bn
W W(a) = an-1 + a0 / 2 W(b) = bn-1 + an-1
K K(a) = an-1 - a0 / 2 K(b) = bn-1 - an-1
1P, W an = P(an-1) + D × n bn = P(bn-1) + D - c0
1P, K an = P(an-1) - D × n bn = P(bn-1) - D + c0
1S, W an = a0 - S(an-1) bn = S(bn-1) + c0 / n
1S, K an = a0 + S(an-1) bn = S(bn-1) - c0 / n
W, K an = an-1 + an-2 bn = bn-1 + bn-2
2P, W an = P1(an-1) + P2(an-1) - W(an-1) bn = b0 * n - |P1(bn-1) - P2(bn-1)|
2P, K an = P1(an-1) + P2(an-1) + K(an-1) bn = b0 * n + |P1(bn-1) - P2(bn-1)|
1P, 1S, W an = max(P(an-1), S(an-1), W(an-1)) bn = P(bn-1) + S(bn-1) - D
1P, 1S, K an = min(P(an-1), S(an-1), K(an-1)) bn = P(bn-1) + S(bn-1) + D
2S, W an = |S1(an-1) - S2(an-1)| + a0 bn = S3(bn-1) - an-2
2S, K an = a0 - |S1(an-1) - S2(an-1)| bn = S3(bn-1) + an-2
1P, W, K an = max(P(an-1), an-1 - an-2) - D bn = max(R(bn-1), G(bn-1), B(bn-1))
1S, W, K an = min(S(an-1), an-1 - an-2) + D bn = min(Y(bn-1), C(bn-1), M(bn-1))

For a-1, use the number of modules on the bomb, modulo 365. For b-1, use the sum of the characters in the serial number (A=1, Z=26), squared, modulo 365.

Simon Stores' manual can be found here.

* For safe digit interactions, the module does NOT detect miliseconds on the countdown timer.

Solving 11-15 Singularity Buttons

Another game was referenced. What was that game again?

Each Singularity Button will show 3 characteristics, the digit visible on the button's screen, the color of the button's surroundings, and the color of the text. The objective is to make all of the Singularity Button's screen stop glowing white by tapping each Singularity Button.

The state of each Singularity Button can be toggled by tapping any Singularity Button that has its screen white. Attempting to tap a Singularity Button that does not have its screen white will result in a strike and not toggle that Singularity Button.

Not only does the tapped Singularity Button toggle itself but other Singularity Buttons will also be toggled if at least 2 properties are shared. If the main button is held for at least 5 timer ticks instead, all Singularity Button's that share at least 2 properties will flash.

Sharable Properties:

  • Button's displayed digit
  • Button's backing color
  • Button's text color
