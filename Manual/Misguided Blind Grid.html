On the Subject of Misguided Blind Grid

A small grid can contain a lot of quirks that may tilt you.

On the module consists of 4 directional buttons and a screen showing a random two digit value in one of many different ways. To disarm the module, navigate through a 5 by 5 grid of distinct values and submit a valid number. The valid number is determined by the starting value and the type of topology that grid is enacting when navigating. More than 1 valid number can occur.

Step 1: Determining the Topology

To determine if the grid wraps around, the defuser will need to come in contact with the edge of the grid. Once walking past the edge of the grid, the following can occur:

  • The defuser will immediately be put back to the last position that is within the grid.
  • The defuser will be wrapped to a different edge of the same grid, with the controls not modified.
  • The defuser will be wrapped to a different edge of the same grid, with the controls rotated or flipped.

This is never easy to determine when a wrap has occurred when navigating. All the defuser can do is to keep walking in one direction until going back to the starting position.

Step 2: Determining the Valid Numbers

With the topology determined, use the starting number on the module to determine what set of numbers are valid from the table underneath. Use the determined topology as the row and the starting number as the column. If there are more than half of the numbers on the grid that share that trait on the intersection or none at all, go down a row until you reach a cell that does not. If you reach the end of that row, go to the other column and continue from there, top to bottom. In the case where none of the valid sets satisfy the condition upon reaching back to the intersection, the valid number is the starting value.
